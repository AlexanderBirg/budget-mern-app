# Budget MERN App

Учебно-дипломное MERN-приложение для бюджетирования процесса разработки ИТ-проекта.

## Что внутри

```text
client/ — React + Tailwind интерфейс
server/ — Express + MongoDB API
server/src/core/ — расчетное ядро
```

## Зачем нужна MongoDB

MongoDB хранит исходные данные расчетной модели:

- проекты;
- задачи проекта;
- исполнителей;
- матрицу `p(g,l)` и `δ(g,l)`;
- сценарии назначений;
- результаты расчетов.

Расчетное ядро не работает напрямую с MongoDB. Сервер достает данные из базы, передает их в ядро, получает результат и сохраняет его обратно.

## Быстрый запуск

Нужны Node.js 18+ и MongoDB.

```bash
npm run install:all
```

Создай файл `server/.env` по примеру `server/.env.example`.

Затем заполни БД демонстрационными данными:

```bash
npm run seed
```

Запусти сервер и клиент:

```bash
npm run dev
```

Открой:

```text
http://localhost:5173
```

Если Firefox плохо открывает `localhost`, используй:

```text
http://127.0.0.1:5173
```

## Что можно редактировать в интерфейсе

В рабочей области проекта доступны:

1. параметры проекта: бюджет, срок, минимальная устойчивость;
2. задачи проекта: название, этап, часы, сложность, критичность и зависимости от других задач;
3. исполнители: роль, грейд, ставка, доступность;
4. матрица модели в табличном виде: `p(g,l)` и `δ(g,l)`;
5. сценарии команды: название, коэффициент слаженности, режим расчета дельты;
6. назначения внутри сценария: какая задача какому исполнителю передана.

После изменения данных нужно нажать кнопку сохранения в соответствующем блоке, затем нажать «Рассчитать сценарии».

## Основной пользовательский сценарий

1. Открыть список проектов.
2. Перейти в демонстрационный проект.
3. При необходимости изменить параметры проекта, задачи, исполнителей, матрицу и сценарии.
4. В блоке сценариев выбрать сценарий и отредактировать назначения задач.
5. Нажать «Рассчитать сценарии».
6. Получить таблицу сравнения, рекомендуемый сценарий и диаграммы Ганта по каждому сценарию.

## Академический смысл приложения

Приложение показывает, что выбор состава команды должен выполняться не только по минимальной стоимости, но и с учетом срока, надежности назначений и риска неопределенности.


## Распараллеливание задач и диаграмма Ганта

В версии v4 задачи поддерживают поле `dependsOnTaskIds`. Оно задает список задач-предшественников: текущая задача может начаться только после их завершения.

Расчетное ядро строит расписание для каждого сценария так:

1. рассчитывает ожидаемую трудоемкость каждой задачи с учетом `p(g,l)`;
2. переводит трудоемкость задачи в длительность с учетом занятости исполнителя;
3. строит календарное расписание по графу зависимостей;
4. формирует массив `schedule`, на основе которого frontend строит диаграмму Ганта.

Это усиливает MVP: различия сценариев видны не только в таблице бюджета, но и в календарном плане выполнения работ.

## Версия v10: автоматический подбор сценария

В этой версии добавлен модуль автоматического подбора сценария команды на основе генетического алгоритма.

### Что делает модуль

Пользователь по-прежнему может вручную создавать сценарии A/B/C, но дополнительно появилась кнопка **«Автоподбор сценария»**. После нажатия сервер:

1. берет проект, задачи, исполнителей, матрицу модели и ограничения из MongoDB;
2. формирует популяцию вариантов назначений исполнителей на задачи;
3. оценивает каждый вариант через расчетное ядро;
4. применяет скрещивание и мутацию назначений;
5. выбирает лучший найденный вариант;
6. сохраняет его как новый сценарий **D — автоматически подобранный**;
7. выполняет повторное сравнение всех сценариев.

### Как это трактовать в дипломе

Один сценарий рассматривается как хромосома, а каждое назначение исполнителя на задачу — как ген. Функция приспособленности учитывает ожидаемый бюджет, срок с учетом неопределенности, индекс устойчивости и штрафы за нарушение ограничений проекта. За счет этого программа не только сравнивает вручную заданные сценарии, но и может сформировать расчетно обоснованный вариант состава команды.

### API

```text
POST /api/projects/:projectId/optimize
```

Тело запроса необязательное. По умолчанию используются параметры:

```json
{
  "populationSize": 48,
  "generations": 70,
  "mutationRate": 0.14,
  "teamFitCoefficient": 0.88,
  "deltaMode": "criticalOnly"
}
```

### Важно

Команда `npm run seed` по-прежнему сбрасывает базу к демонстрационным данным. Перед экспериментами с автоподбором лучше сделать бэкап MongoDB.
